## Using the REAL HLSDataset

The real dataset (~50 GB) is public:

```
https://github.com/UT-LCA/ML4Accel-Dataset/tree/main/fpga_ml_dataset
```

Steps to swap in real data:

1. Clone the repo above
2. For each benchmark, locate `post_hls_info.csv` and `post_impl_info.csv`
3. Concatenate them:

```python
import pandas as pd, glob
hls_files = glob.glob('fpga_ml_dataset/**/post_hls_info.csv', recursive=True)
hls_all   = pd.concat([pd.read_csv(f) for f in hls_files], ignore_index=True)
hls_all.to_csv('data/raw/post_hls_info.csv', index=False)

impl_files = glob.glob('fpga_ml_dataset/**/post_impl_info.csv', recursive=True)
impl_all   = pd.concat([pd.read_csv(f) for f in impl_files], ignore_index=True)
impl_all.to_csv('data/raw/post_impl_info.csv', index=False)
```

4. Update column names in `src/features.py` to match the real CSV headers
5. Re-run `python src/train.py`

## Paper methodology (Case Study 2)

| Item | Value |
|------|-------|
| Dataset | Machsuite + Polybench subsets |
| FPGA | ZU9EG |
| Samples | ~8,735 |
| Test split | Random 20% |
| Cross-validation | 10-fold; 75% train / 25% val within fold |
| Metric | MAPE (Mean Absolute Percentage Error) |
| Models | XGBoost (single-task), Lasso (single-task) |
| Targets | #LUT, #FF, #DSP, #BRAM post-implementation |

### Table VI (paper results)

| Model | LUT | FF | DSP | BRAM |
|-------|-----|----|-----|------|
| HLS Estimate | 63.2% | 34.1% | 0.0% | 1.8% |
| XGB | **3.2%** | **2.3%** | NA | **0.1%** |
| Lasso | 13.2% | 15.4% | NA | NA |

> Note: results differ from the paper's exact numbers because the synthetic data
> approximates (but does not perfectly replicate) the real dataset statistics.
> Swap in the real CSVs to reproduce the paper's numbers.

---

## Key finding

The HLS tool grossly overestimates LUT (63.2% error) and FF (34.1% error)
because it simply sums functional unit contributions without modeling
on-chip resource optimization. XGBoost reduces LUT error to **3.2%**
and FF error to **2.3%** by learning the mapping from HLS estimates and
operator counts to post-implementation actuals.
